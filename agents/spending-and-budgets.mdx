**Goal:** an agent that can spend — inside a budget it cannot exceed and you can revoke at any time. You need a [connected agent](/agents/connect) first.

## How agents pay for things

Agents on Perflo pay for third-party services per call using an open standard called x402: a service quotes a price per request, and the agent pays exactly that, instantly, from your balance. (Settlement happens in digital dollars — USDC or USDT — on the Base network, with Tempo as a second rail; your agent handles all of it.) No subscriptions, no API keys, no cards on file with dozens of vendors. Your agent pays cents for exactly what it uses.

## The service marketplace

Perflo curates a marketplace of {marketplaceSize} that an agent can use out of the box: web search, deep research, email sending, image and video generation, phone calls, and a large catalog of data and automation services. Services are probe-verified before listing, and your agent can also describe a task in plain language and let Perflo pick the best vendor, with automatic failover if one fails. Browse the categories on [The service marketplace](/agents/services).

## Budgets

Every spending session you grant has:

- caps per hour, per day, and per month,
- a recipient allowlist (services and addresses),
- asset restrictions,
- an expiry date (30 days default, 90 maximum).

Every payment is checked against the budget before it is approved — and cumulative caps count money still in flight, so concurrent requests cannot race past a cap together. Denials return a stable machine-readable code to the agent and appear in your activity feed; every code is documented on [Errors and denials](/agents/errors-and-denials).

## Auto top-up, your way

You can create a rule that refills an agent's budget from your balance or Earn when it runs low, but only within a monthly ceiling you define. The ceiling cannot be raised by the agent.

## The kill switch

Say "stop agent spending" to your assistant, tap the switch in the app, or revoke the session in settings. All agent spending stops immediately, everywhere, and stays stopped until you re-enable it. Re-enabling is covered in [Agent troubleshooting](/agents/troubleshooting).

## Statements

Agent spending is itemized like a card statement: which service, which agent session, what price, when. Monthly statements make reconciliation straightforward for businesses.

## If something goes wrong

- **The agent reports a denial:** match the code on [Errors and denials](/agents/errors-and-denials) — most fixes are one setting in the app.
- **The agent stopped mid-task:** expiry, cap, or kill switch — see [Agent troubleshooting](/agents/troubleshooting).

<Tip>
A good starter budget: a small daily cap, five allowlisted services, 30-day expiry. Expand once you have watched a week of activity.
</Tip>

<Note>
Marketplace services are operated by third parties, and agent spending is real spending — budgets cap how much can go out, not whether a paid result is useful. Agent activity is screened and monitored exactly like human activity.
</Note>
